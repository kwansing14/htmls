Run NanoClaw with a free local AI on a Raspberry Pi

A pocket-sized, always-on AI agent. Lives on a Pi in your home network, sips power, runs 24/7, and never sends a byte to a cloud API. Pair NanoClaw (the slim ARM build of OpenClaw) with Ollama and a small open-weight model.

📅 May 2026 ⏱ ~15 min read 🍓 Raspberry Pi 5 💸 Hardware: ~$120 · LLM cost: $0
What you're building

The stack

NanoClaw is the trimmed-down ARM build of OpenClaw — same agent loop, fewer optional integrations, designed to run on a Pi as a long-lived daemon. It talks to Ollama on localhost:11434 using the OpenAI-compatible API, so swapping the model later (or pointing at a beefier home server) is a one-line config change.

Pick your Pi

Which Raspberry Pi?

The model and the RAM size both matter — and they matter in that order. A Pi 5 is roughly 2–3× faster than a Pi 4 at LLM inference thanks to a newer CPU, faster RAM, and a wider memory bus. Below 8 GB RAM you're stuck with sub-1B models that struggle with tool calling.

Pi modelBest model to runSpeed · Notes
Pi 5 · 16 GB recommended Qwen 2.5 3B Instruct ~4–6 tok/sec. Headroom for OS + agent + browser tools. The sweet spot.
Pi 5 · 8 GB Qwen 2.5 3B Instruct ~4–6 tok/sec. Same speed, less headroom. Fine if you don't run a desktop on it.
Pi 4 · 8 GB Llama 3.2 3B Instruct (Q4) ~1.5–2.5 tok/sec. Usable for short prompts. Painful for agent loops.
Pi 4 · 4 GB / Pi Zero 2 W Qwen 2.5 0.5B Instruct Tiny, weak at tool use. More of a toy than an agent. Skip.

Before you start

Raspberry Pi reality check

Heads up: The Pi is great as an always-on, low-power agent — not a fast one. Expect 4–6 tokens/sec on a 3B model, ~10 seconds before the first token appears. That's bad for chat, fine for cron-triggered jobs and webhooks. Treat it like a tiny home server, not a desktop assistant.
Setup

Step-by-step

Flash Raspberry Pi OS Lite (64-bit)

Use Raspberry Pi Imager. Pick the 64-bit Lite image — no desktop, more RAM for the model. In the Imager's settings (gear icon), preconfigure SSH, hostname (nanoclaw), and Wi-Fi.

shell# after first boot, ssh in from your laptop ssh pi@nanoclaw.local # confirm you're on a 64-bit kernel uname -m # should print: aarch64 # update everything sudo apt update && sudo apt full-upgrade -y

Install Node 24 via NodeSource

The apt default is too old. Use NodeSource's ARM64 repo.

shellcurl -fsSL https://deb.nodesource.com/setup_24.x | sudo -E bash - sudo apt install -y nodejs node --version # v24.x.x

Install & start Ollama

Ollama has a native ARM64 build. The installer registers it as a systemd service so it survives reboots — exactly what you want for a Pi.

shellcurl -fsSL https://ollama.com/install.sh | sh # confirm the service is running systemctl status ollama curl http://localhost:11434/api/tags

Pull a model

Qwen 2.5 3B Instruct is the recommended pick — best tool-calling-per-byte for Pi 5 RAM budgets. Pull happens once and is cached on disk.

shellollama pull qwen2.5:3b-instruct # on a Pi 4 (8 GB)? this is slower but lighter: # ollama pull llama3.2:3b-instruct-q4_K_M # smoke test — expect ~10s before first token ollama run qwen2.5:3b-instruct "in 5 words, say hi"

Install NanoClaw

Global npm install. NanoClaw ships an ARM64 binary; the install script picks the right one automatically.

shellsudo npm install -g nanoclaw@latest # one-time setup — installs the systemd unit nanoclaw onboard --install-daemon --user pi

Point NanoClaw at Ollama

Edit ~/.nanoclaw/nanoclaw.json. Lower the agent's context window — keeps memory pressure down and avoids swap on smaller Pi boards.

json5{ providers: { ollama: { type: "openai-compatible", baseURL: "http://localhost:11434/v1", apiKey: "ollama" // dummy — Ollama doesn't check it } }, agent: { model: "ollama/qwen2.5:3b-instruct", contextWindow: 4096, // keep small on Pi maxIterations: 8 // stop runaway loops } }

Smoke test & expose it on the LAN

Try a tiny tool-using task. Then bind NanoClaw to 0.0.0.0 so you can hit it from your laptop or trigger it via webhook.

shellcd ~ nanoclaw "list the 5 largest files in my home directory" # expose the HTTP trigger on the LAN (port 7878 by default) nanoclaw serve --host 0.0.0.0 --port 7878 # from your laptop: curl -X POST http://nanoclaw.local:7878/run \ -H "content-type: application/json" \ -d '{"prompt":"summarize today/'"'"'s news headlines"}'

When things break

Troubleshooting

Random reboots under load
Almost always undervoltage. Use the official 27 W PSU, not a phone charger. Check vcgencmd get_throttled — anything non-zero = power or thermal issue.
Inference crawls / fans max out
Pi 5 throttles at 80°C. Install the official Active Cooler if you haven't. Drop to a smaller model (qwen2.5:1.5b-instruct) or a heavier quantization (q4_0).
Out of memory during pull or first run
Make sure swap is enabled and at least 2 GB: edit /etc/dphys-swapfile, set CONF_SWAPSIZE=2048, then sudo systemctl restart dphys-swapfile.
Tool calls don't fire
Make sure the model name ends in -instruct. Pi-sized base models can't tool-call reliably. Qwen 2.5 Instruct and Llama 3.2 Instruct both work.
SD card wears out / corruption
Move Ollama's model dir to an NVMe SSD or USB SSD: stop the service, mv ~/.ollama /mnt/ssd/ollama, symlink it back. SD cards aren't built for the write volume.
Need more speed than the Pi gives
Same NanoClaw config, different baseURL: point it at a free hosted endpoint (Groq, Gemini free tier, OpenRouter free models). Pi becomes the always-on orchestrator, not the inference box.
