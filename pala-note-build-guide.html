01 How it works

The whole idea is friction-free capture. You press a button, speak your thought, and let go. That's the entire interaction.

Under the hood, the device is built around a single, clever board: the Waveshare ESP32-S3 1.54" e-Paper development board. The flow looks like this:

  • Capture — Holding the record button writes your voice to a .wav file on a microSD card. No internet needed in the moment.
  • Wait — The e-ink display shows your notes and stays lit with zero power draw. E-ink only uses energy when the image changes, which is why the battery lasts so long.
  • Sync — On a schedule (or when you press sync), the device connects to Wi-Fi and uploads recordings to OpenAI's Whisper API.
  • Transcribe — Whisper returns clean text, which is saved back to the SD card as .txt and shown on the screen.
  • Review — A tiny web dashboard served by the device itself lets you read and organize notes from your phone or laptop.
Why not transcribe on-device? The ESP32-S3 doesn't have the horsepower to run speech-to-text locally. Offloading to Whisper keeps the device cheap, tiny, and power-efficient — at the cost of needing Wi-Fi and an OpenAI API key.

03 Tools you'll need

  • A USB-C cable (data, not charge-only) to flash and power the board.
  • A computer with the Arduino IDE or PlatformIO/ESP-IDF installed.
  • Access to a 3D printer, or a print service like Craftcloud / JLCPCB / a local library makerspace.
  • An OpenAI account with an API key and a little credit on it.
  • Optional: tweezers and a plastic spudger for tucking the battery in cleanly.

05 Assembly, step by step

  1. Test the bare board first

    Before any soldering or casing, plug the board into USB-C and confirm it powers on and the e-ink panel refreshes. Flash a "hello world" sketch (see Section 06) so you know the hardware is good.

  2. Format and insert the microSD card

    Format the card as FAT32, then slot it into the board's microSD reader. This is where every recording and transcript will live.

  3. Connect the speaker

    Plug the small speaker into the board's MX1.25 speaker header. If your board didn't come with one, this is the cheap add-on from the parts list.

  4. Attach the LiPo battery

    Connect the 500 mAh LiPo to the battery connector. Mind the polarity — a reversed connector can damage the board. Charging happens automatically over USB-C.

  5. Seat everything in the case

    Lay the battery flat, route the speaker wire, and snap the board into the printed case. Lagier's design clicks together with no screws. Make sure the buttons line up with their cutouts.

  6. First boot

    Power on. You should see the dashboard/idle screen on the e-ink panel. Press record, speak, release — a .wav file should appear on the SD card.

06 Flashing the firmware

If you bought the files, you'll be flashing Lagier's prebuilt firmware — just point your flasher at the board and go. If you're rolling your own, set up your toolchain for the ESP32-S3.

Using the Arduino IDE:

  • Install the ESP32 board package from Espressif via Boards Manager.
  • Select "ESP32S3 Dev Module" as your board.
  • Enable PSRAM and set Flash size to match your board.
  • Add libraries for the e-ink panel (Waveshare's GxEPD2-style driver), SD, WiFi, and an I²S audio library.

The Waveshare wiki has board-specific demo code and pin maps — start there so the display and audio pins are correct:

📖 Waveshare wiki — ESP32-S3-ePaper-1.54 demos & pinout waveshare.com/wiki/ESP32-S3-ePaper-1.54 ↗

Flash, then open the serial monitor at 115200 baud to watch it boot and connect to Wi-Fi.

07 Wiring up Whisper (the AI part)

This is what makes it an "AI" note device. The board records audio locally, then ships it to OpenAI's Whisper speech-to-text model for transcription.

  1. Get an OpenAI API key

    Create a key at the OpenAI API keys page and add a few dollars of credit. Whisper is billed per minute of audio and is very cheap for personal use.

  2. Put your key and Wi-Fi in the config

    In the firmware's config (or a config.h / settings file on the SD card), set your Wi-Fi SSID, password, and the API key. Never share your key publicly.

  3. Understand the upload call

    On sync, the device POSTs each .wav file as multipart form data to the transcription endpoint and gets back text. The shape of the request:

POST https://api.openai.com/v1/audio/transcriptions
Authorization: Bearer YOUR_API_KEY
Content-Type: multipart/form-data

file=@note_0007.wav
model=whisper-1
response_format=text

The returned text gets written next to the recording as note_0007.txt and rendered on the e-ink screen. See the official guide for parameters and limits:

🗣️ OpenAI — Speech-to-Text (Whisper) guide platform.openai.com/docs/guides/speech-to-text ↗
Privacy note Audio leaves the device and is sent to OpenAI for transcription. If that matters to you, consider a self-hosted Whisper server (e.g. whisper.cpp on a home machine) and point the device at that endpoint instead.

08 The 3D-printed case

The case is what turns a dev board into something you'd actually carry. Lagier's design is a snap-together, screwless enclosure included in the Ko-fi bundle.

  • Print in PLA or PETG at 0.2 mm layer height — no supports if oriented correctly.
  • Leave room for the battery to sit flat behind the board.
  • Test-fit the buttons before final assembly; tweak tolerances if your printer runs tight.
  • No printer? Upload the STL to a print-on-demand service or a local makerspace.

09 Going fully DIY (the harder path)

Want to build the firmware yourself instead of buying it? Here's the architecture to recreate:

  • Audio capture: read the I²S microphone into a buffer while the record button is held; write a WAV header + PCM samples to the SD card.
  • Display: drive the e-ink panel only on state changes — idle screen, "recording…", note list, transcript view.
  • Storage: use the SD library with incrementing filenames so notes never overwrite each other.
  • Sync: a Wi-Fi task that walks the SD card for untranscribed .wav files, uploads each to Whisper, and writes back the .txt.
  • Dashboard: run a small WebServer on the ESP32 that lists notes from the SD card so you can read them in a browser on the same network.
  • Power: use deep-sleep between interactions; wake on the button GPIO. This is the key to multi-day battery life.

Honestly, for $6.50 the prebuilt firmware saves a lot of evenings — but rebuilding it is a fantastic way to learn ESP32, I²S audio, and e-ink.
