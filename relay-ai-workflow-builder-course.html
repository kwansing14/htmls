01

The video

One video, an entire SaaS: Code With Antonio's free full-course format applied to the hottest 2026 topic — browser-driving AI agents.

The video is the YouTube release of Antonio's course “Build an AI Workflow Builder like n8n” (published mid-July 2026). Depending on when you loaded it, YouTube may have shown you an A/B-tested title — either “Build an AI Workflow Builder like N8N | Agentic Coding” or “How to Build AI Agents That Control Real Browsers | Full Stack SaaS Course 2026”. Same video.

It follows his usual long-form build formula: start from an empty folder, end with a deployed production app. The twist this time is twofold: the app itself is an automation platform (an n8n clone he calls Relay), and the course workflow leans on agentic coding — an AI coding agent is set up in chapter 2 and used throughout the build.

▎i

The course's own pitch line: give the agent a goal like “configure a Porsche 911 and email me the link” — then watch a cloud browser click through the configurator and a workflow email you the result.

02

What gets built

Relay is a complete AI workflow automation platform — not a toy demo. Five pillars make up the product.

feat / canvas

Drag-and-drop canvas

An n8n-style node editor built with React Flow: triggers, action nodes, connections, a toolbar, and custom node UIs.

feat / agent

Browser agent node

A node that takes over a real Chrome browser in the cloud and autonomously clicks, scrolls, and reasons its way to a goal.

feat / runs

Durable executions

Every run is a background job with live, node-by-node status streamed back onto the canvas, plus a console panel and session replay.

feat / collab

Real-time multiplayer

Live cursors, names, and avatars on the canvas — multiple people editing the same workflow at once.

feat / saas

The SaaS shell

Auth, organizations, and billing; transactional email; error monitoring; and a production deployment at the end.

Workflows persist to Postgres, belong to organizations, and are executed server-side — so the finished product is architecturally honest: the canvas is just the editor, and the real work happens in a durable job queue.

03

The agent node

The headline feature: an Agent node backed by Browserbase and Stagehand, driving a real Chrome session in the cloud.

Instead of scripting brittle selectors, the Agent node gets a natural-language goal. Stagehand (an AI browser-automation framework) plans and performs the actions — click, scroll, type, extract — while Browserbase hosts the actual headless-but-real Chrome session remotely.

agent-node — the idea, distilled
// what the Agent node effectively does per run
const session = await browserbase.createSession()   // real Chrome, in the cloud
const agent   = stagehand.agent({ session })

await agent.execute("configure a Porsche 911 and grab the share link")
// → agent decides each step itself: navigate, click, scroll, extract

return { output: agent.result, replayUrl: session.replay() }  // nothing is a black box

Because it's a real browser session, every run can be replayed: you watch exactly what the agent saw and did, frame by frame. The course wires that replay into Relay's UI (chapter 22), which turns "trust me, the agent did it" into an auditable recording.

▎!

Both Browserbase and Stagehand are paid/hosted-tier services (free tiers exist). Following along end-to-end means creating accounts for roughly nine external services — budget setup time for that.

04

Execution engine

Runs are durable background jobs on Trigger.dev — the canvas only watches.

When you hit run, Relay hands the workflow to Trigger.dev, which executes it as a durable job: long-running, retryable, and independent of the browser tab that started it. The course builds out:

  • Node-by-node live status — execution state streams back to the canvas in real time, lighting up each node as it runs (chapter 18).
  • Data passthrough — outputs of one node become inputs of the next, the core mechanic of any workflow engine (chapter 17).
  • A console panel — logs and run output surfaced inside the app, not just in a dashboard elsewhere (chapter 21).
  • An email node — sends via Resend, closing the loop on the "…and email me the link" demo (chapter 20).

This is the part of the course with the most transferable architecture: the trigger → durable job → live status → replay pipeline applies to almost any AI product, not just workflow builders.

05

Multiplayer canvas

The canvas is fully collaborative via Liveblocks — a Figma-style presence layer over React Flow.

Three chapters (11–13) integrate Liveblocks: authenticating users into rooms, then rendering live cursors, names, and avatars for everyone editing a workflow. Combined with Clerk organizations, you get the standard B2B collaboration story — teammates in the same org share and co-edit workflows in real time.

It's a deliberately thin slice of multiplayer (presence rather than full CRDT document sync), which keeps the chapter count honest while still demoing well.

06

The stack

A very 2026 stack: managed services for everything, glue code in Next.js.

LayerToolRole in Relay
frameworkNext.js + ReactApp shell, dashboard, API routes
canvasReact FlowNode editor: triggers, actions, custom nodes
agentBrowserbase + StagehandCloud Chrome sessions + AI browser control
jobsTrigger.devDurable workflow executions, live run status
realtimeLiveblocksMultiplayer presence: cursors, names, avatars
auth+billingClerkAuth, organizations, subscription billing
databaseNeon + DrizzleServerless Postgres, typed schema for workflows
emailResendTransactional email + the workflow email node
observabilitySentryError monitoring in production
▎i

Notice what's absent: no self-hosted queue, no websocket server, no ORM migrations by hand-rolled SQL. Every hard infrastructure problem is delegated to a service — the course is as much about composing managed platforms as about writing code.

08

Who it's for

Intermediate React/Next.js developers who want one project that touches every 2026 buzzword — legitimately.

  • Watch it if you want to understand how browser-driving agents actually ship inside a product: session hosting, replay, durable jobs, and live status — not just a Stagehand hello-world.
  • Watch it if you're building any workflow/automation tool; the React Flow + Trigger.dev + data-passthrough pattern is the reference architecture.
  • Skip it if you want deep fundamentals — with nine managed services, much of the course is integration work, and pricing/vendor lock-in trade-offs are yours to weigh.
  • Prerequisites: comfortable React + TypeScript; Next.js familiarity helps. Everything else is introduced service by service.

The pitch that sums up the whole build: type a goal in plain English, watch a real browser in the cloud accomplish it, replay every step it took, and bill for it — all on a canvas your team edits together.
