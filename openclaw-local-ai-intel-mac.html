Setup guide

Run OpenClaw with a free local AI on an Intel MacBook

A self-hosted personal AI agent with zero API bills. Uses Ollama as the local LLM backend so the whole stack runs on your laptop, no cloud, no credit card.

📅 May 2026 ⏱ ~10 min read 💻 Intel MacBook 💸 Total cost: $0
What you're building

The stack

OpenClaw is the agent — a Node.js app that takes a prompt, decides which tools to call (filesystem, shell, browser, messaging apps), and loops until the task is done. By default it talks to a paid LLM API. We're swapping that out for Ollama running a small open-weight model on your Mac, which speaks the OpenAI API format so OpenClaw can use it without code changes.

Before you start

Intel Mac reality check

Heads up: Intel Macs don't have Apple Silicon's unified-memory speed advantage. Expect 3–8 tokens/sec on a 7B model running on CPU. That's fine for chat and short tasks, slow for long agent loops. RAM is the constraint that decides which model you can run.
Your RAMRecommended modelNotes
8 GB Llama 3.2 3B Instruct Only thing that fits. Weak at multi-step agent work.
16 GB Qwen 2.5 7B Instruct recommended Best small-model tool calling. Sweet spot for this setup.
32 GB+ Qwen 2.5 14B Instruct Noticeably smarter, noticeably slower on CPU.
Setup

Step-by-step

Install Node 24

OpenClaw requires Node 24 (or 22.16+). Use Homebrew if you don't have it.

shell# install Homebrew if needed /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)" # install Node brew install node@24 node --version # should print v24.x.x

Install & start Ollama

Ollama runs the local model and exposes an OpenAI-compatible API at http://localhost:11434/v1.

shellbrew install ollama # start Ollama as a background service brew services start ollama # confirm it's listening curl http://localhost:11434/api/tags

Pull a model

Qwen 2.5 7B Instruct is the recommended pick — strong tool calling for its size. ~5 GB download.

shellollama pull qwen2.5:7b-instruct # 8 GB RAM? use this instead: # ollama pull llama3.2:3b-instruct # quick smoke test ollama run qwen2.5:7b-instruct "say hi in 5 words"

Install OpenClaw

Global install via npm, then run the onboarding wizard once to set up the daemon.

shellnpm install -g openclaw@latest # one-time setup — installs the background daemon openclaw onboard --install-daemon

Point OpenClaw at Ollama

Edit ~/.openclaw/openclaw.json (JSON5 — comments allowed). Add an Ollama provider and set the agent model. Exact key names may vary slightly between versions; if the agent doesn't pick up the endpoint, check docs.openclaw.ai/gateway/configuration.

json5{ providers: { ollama: { type: "openai-compatible", baseURL: "http://localhost:11434/v1", apiKey: "ollama" // dummy — Ollama doesn't check it } }, agent: { model: "ollama/qwen2.5:7b-instruct" } }

Smoke test it

Ask the agent to do something tiny that touches a tool (the filesystem). If it lists your files, the wiring is working.

shellcd ~/Documents openclaw "list the 5 most recently modified files in this folder"
What it's actually good for

5 simple use cases

Local 7B models are not GPT-5. Pick tasks that are narrow, structured, and short-loop — where the agent runs a few tools, transforms text, and stops. Skip anything needing deep reasoning over huge contexts.

When things break

Troubleshooting

Slow / hangs mid-loop
Drop to llama3.2:3b-instruct, or switch the provider to a free hosted endpoint — Groq, Google AI Studio's Gemini free tier, or OpenRouter's free models. Same config, different baseURL and apiKey.
Tool calls don't fire
Make sure the model name ends in -instruct or -chat. Base models can't do tool calling. Qwen 2.5 Instruct and Llama 3.x Instruct both work; Mistral 7B base does not.
Out of memory / fans spin up
Use the Q4_K_M quantization (default in Ollama). Close Chrome. Reduce the agent's context window in openclaw.json. If you have a discrete AMD GPU on a 16″ MBP, llama.cpp with the Metal backend is faster than CPU-only Ollama.
"Cannot find module 'openclaw'"
Your global npm prefix isn't on $PATH. Run npm config get prefix and add $(npm config get prefix)/bin to your shell rc file.
Model gives up / refuses tools
Smaller models follow shorter, more concrete prompts better. Replace "help me with X" with "do step 1, then step 2, then stop." Less open-ended = better results.
